# CRITICAL: MDX Formatting Rules

**You are generating MDX (Markdown + JSX) that MUST compile with @mdx-js/mdx.**

## 1. THE "RAW TEXT" PREVENTION RULES (HIGHEST PRIORITY):

* **ZERO INDENTATION for Root Tags:**
    * ❌ **BAD:** Indenting `<details>` or `<div>` by 4 spaces or a tab. (This turns it into a code block).
    * ✅ **GOOD:** Start `<details>` and `<div>` tags at the very beginning of the line (Column 0).

* **VERTICAL SPACING (The "Empty Line" Rule):**
    * You MUST place a blank empty line **before** and **after** every block-level JSX tag (`<details>`, `<div>`, `<summary>`).

## 2. STRICT SYNTAX REQUIREMENTS:

1.  **JSX must be valid React JSX and fully closed**
    * All tags must have closing tags: `<details>...</details>`
    * Self-closing tags need `/`: `<Component />`
    * No partial or dangling JSX

2.  **Do NOT nest Markdown inside JSX children**
    * ❌ BAD: `<div># Heading</div>`
    * ✅ GOOD: Use pure HTML inside JSX (e.g., `<ul>` instead of `- list`), or keep Markdown outside JSX.

3.  **Do NOT place JSX inside Markdown lists, tables, or blockquotes**
    * ❌ BAD: `- Item with <Component />`
    * ✅ GOOD: Use JSX before/after the list, not inside.

4.  **No HTML comments**
    * ❌ BAD: ``
    * ✅ GOOD: Use JSX comments `{{/* comment */}}` if needed.

## 3. ESCAPE SPECIAL CHARACTERS IN TEXT:

1.  **Curly braces** → Use backticks or escape
    * ❌ BAD: `{{key: value}}`, `{{}}`, `{{x}}`
    * ✅ GOOD: `` `{{key: value}}` `` or `\{{key: value\}}`

2.  **Angle brackets `<` and `>`** → Use HTML entities
    * ❌ BAD: `<10ms`, `x < 5`, `value > max`, `a <- b`
    * ✅ GOOD: `&lt;10ms`, `x &lt; 5`, `value &gt; max`, `a &lt;- b`

3.  **Horizontal rules `---`** → Use alternative syntax
    * ❌ BAD: `---` (reserved for frontmatter)
    * ✅ GOOD: `* * *` or `___`

**Note:** Inside fenced code blocks (` ```python `), you do NOT need to escape `<`, `>`, or `{{}}`.

**Syntax Safety Rules (CRITICAL for Rendering):**

2.  **Flow Type:** Prefer `flowchart TD` or `flowchart LR` (Left-to-Right) over `graph`.
3.  **Label Safety:** You **MUST** enclose text labels in double quotes.
    * ❌ BAD: `A[User Login]` (Breaks if special chars exist)

4.  **ID Safety:** Node IDs must be alphanumeric strings without spaces.
    * ❌ BAD: `User Input-->Database`
